KEY FEATURES
============

✓ Two-stage pipeline (cheap local + powerful remote)
✓ Budget tracking and enforcement
✓ Graceful degradation if components unavailable
✓ Configurable thresholds and models
✓ Batch processing for efficiency
✓ Structured logging with tracing
✓ Strong type safety with enums and UUIDs
✓ Database persistence of scores, summaries, tags
✓ Zero new dependencies (uses existing Cargo.toml)
✓ Production-ready error handling

DEPENDENCIES USED
=================

All from existing Cargo.toml:
- anyhow (error handling)
- async-trait (async traits)
- chrono (timestamps)
- serde / serde_json (serialization)
- reqwest (HTTP client with rustls-tls)
- tracing (structured logging)
- tokio (async runtime)
- uuid (ID generation)
- sqlx (database)

DATABASE SCHEMA
===============

Pre-existing tables that intelligences pipeline populates:

scores table:
  id TEXT PRIMARY KEY
  item_id TEXT FOREIGN KEY
  interest_name TEXT
  score REAL
  reasoning TEXT
  model_used TEXT
  scored_at TEXT

summaries table:
  id TEXT PRIMARY KEY
  item_id TEXT FOREIGN KEY
  summary TEXT
  model_used TEXT
  created_at TEXT

tags table:
  item_id TEXT FOREIGN KEY
  tag TEXT
  PRIMARY KEY (item_id, tag)
